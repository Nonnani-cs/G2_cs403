# รายงานการวิเคราะห์ข้อบกพร่องของระบบ (Codebase Error Analysis Report)

รายงานฉบับนี้รวบรวมข้อบกพร่องที่พบจากการตรวจสอบซอร์สโค้ดทั้งหมดในไดเรกทอรี `Projest` โดยแบ่งหมวดหมู่ตามประเภทของปัญหา

---

## 1. ข้อผิดพลาดระดับวิกฤต (Critical Errors)

### 1.1 การใช้เส้นทางไฟล์แบบระบุเจาะจง (Hardcoded Absolute Paths)
*   **ไฟล์ที่พบ:** ทุกสคริปต์ Python ในโปรเจกต์ (เช่น `replace_drugs_v3.py`, `add_dispense_drug_list.py`, `update_receive_logic.py`, ฯลฯ)
*   **บรรทัดที่พบ:** มักอยู่ช่วงบรรทัดที่ 3-10 ของทุกไฟล์ Python
*   **สาเหตุ:** มีการใช้ Path เช่น `filepath = '/Users/atcharapornn/Desktop/Projest/...'` ซึ่งเป็นตำแหน่งไฟล์ในเครื่องส่วนตัวของผู้พัฒนาคนก่อน
*   **ผลกระทบ:** สคริปต์จะ **รันไม่ผ่านทันที (FileNotFoundError)** เมื่อนำไปรันในเครื่องอื่นหรือ OS อื่น (เช่น Windows) ทำให้ระบบอัตโนมัติทั้งหมดใช้งานไม่ได้

---

## 2. ข้อผิดพลาดด้านตรรกะและการทำงาน (Logic Errors)

### 2.1 การแก้ไขโค้ดด้วย Regex ที่เปราะบาง (Fragile Regex Injection)
*   **ไฟล์ที่พบ:** สคริปต์ตระกูล `fix_*.py` และ `add_*.py` ทั้งหมด
*   **สาเหตุ:** ใช้ `re.sub` ค้นหาข้อความ HTML ที่เป็น Hardcode ยาวๆ เพื่อแทนที่ เช่นการหา `div` ที่ซ้อนกันหลายชั้น
*   **ผลกระทบ:** หากมีการแก้ไขไฟล์ HTML เพียงเล็กน้อย (เช่น เผลอกด Space หรือ Newline) Regex จะหาตำแหน่งไม่เจอ ทำให้สคริปต์รันผ่านแต่ **ไม่เกิดการเปลี่ยนแปลงใดๆ** ในไฟล์ หรืออาจทำให้โครงสร้าง HTML พัง (เช่น ลืมปิด `</div>`)

### 2.2 พารามิเตอร์ของฟังก์ชันไม่สอดคล้องกัน (Function Signature Mismatch)
*   **ไฟล์ที่พบ:** `replace_drugs_v3.py` เชื่อมโยงกับ `manage.html`
*   **สาเหตุ:** 
    *   ใน Python สร้างแถวตารางที่เรียก `onclick="selectDrug(this, 'CODE', 'NAME')"` (3 ค่า)
    *   แต่ใน `manage.html` บรรทัด 490 ฟังก์ชัน `selectDrug(row, code)` รับเพียง 2 ค่า
*   **ผลกระทบ:** ข้อมูล `name` จะถูกส่งเข้าไปแต่ฟังก์ชันรับไม่ได้ หรือหากมีการปรับแก้ JS ในภายหลังอาจทำให้เกิด Bug ในการแสดงผลชื่อยาผิดพลาด

### 2.3 ฟังก์ชันที่จำเป็นหายไป (Missing Essential Functions)
*   **ไฟล์ที่พบ:** `patients.html` (บรรทัด 348) และ `status.html` (บรรทัด 257)
*   **สาเหตุ:** มีการเรียกใช้ฟังก์ชัน `renderPatientTable()` และ `renderInventoryTable()` ใน `window.onload` แต่ **ไม่พบการประกาศฟังก์ชัน** เหล่านี้ในไฟล์ HTML นั้นๆ
*   **ผลกระทบ:** เมื่อเปิดหน้าเว็บจะเกิด Error `Uncaught ReferenceError: renderPatientTable is not defined` ทำให้หน้าจอว่างเปล่าและไม่แสดงข้อมูลผู้ป่วยหรือสต็อกยา

### 2.4 การอ้างอิง ID ที่ไม่มีอยู่จริง (Missing DOM Element IDs)
*   **ไฟล์ที่พบ:** `move_search_and_revert.py` และ `receive.html`
*   **สาเหตุ:** สคริปต์พยายามหา ID `order-list-tbody` เพื่อแก้ไข แต่ในไฟล์ `receive.html` ตั้งต้นบางเวอร์ชันอาจไม่มี ID นี้
*   **ผลกระทบ:** การทำงานของปุ่มเลือกยาจะไม่สามารถอัปเดตข้อมูลลงตารางด้านล่างได้ (Preview Logic ล้มเหลว)

## 3. ข้อผิดพลาดด้านข้อมูลและความปลอดภัย (Security & Data Errors)

### 3.1 ระบบเซสชันไม่ปลอดภัย (Insecure Session Management)
*   **ไฟล์ที่พบ:** `login.html`, `index.html`, และไฟล์อื่นๆ ที่มีส่วน Check Session
*   **สาเหตุ:** ใช้ `localStorage.getItem('currentUser')` ในการตรวจสอบสิทธิ์
*   **ผลกระทบ:** ผู้ใช้งานที่มีความรู้ด้าน Technical สามารถสร้าง Object จำลองใน Console ของ Browser เพื่อ **ปลอมตัวเป็น Senior Pharmacist** ได้โดยไม่ต้องรู้รหัสผ่านจริง

### 3.2 ข้อมูล MasterData เป็นค่าว่าง
*   **ไฟล์ที่พบ:** `manage.html` (บรรทัด 462) และ `receive.html`
*   **สาเหตุ:** ตัวแปร `masterData` และ `lotData` ถูกประกาศเป็น `{}` หรือ `[]` ว่างๆ ไว้
*   **ผลกระทบ:** หากไม่มีสคริปต์ Python รันเพื่ออัดข้อมูล (Inject) เข้าไปก่อน หน้าเว็บจะแสดงรายละเอียดเป็น "Unknown" หรือค่าว่างเสมอเมื่อคลิกเลือกรายการ

---

## 4. ปัญหาด้านการบำรุงรักษา (Maintainability Issues)

### 4.1 ความซ้ำซ้อนและเวอร์ชันที่สับสน (Version Fragmentation)
*   **ไฟล์ที่พบ:** `replace_drugs.py`, `replace_drugs_v2.py`, `replace_drugs_v3.py`
*   **สาเหตุ:** มีการสร้างไฟล์ใหม่ทุกครั้งที่มีการอัปเดตแทนการใช้ระบบ Version Control (Git) อย่างถูกต้อง
*   **ผลกระทบ:** ยากต่อการตรวจสอบว่าไฟล์ไหนคือเวอร์ชันล่าสุดที่ใช้งานจริง (Single Source of Truth) และอาจนำไฟล์เก่าไปรันจนข้อมูลถอยหลัง (Regress) ได้

---

## สรุปข้อเสนอแนะในการแก้ไข:
1.  ปรับเปลี่ยน Path ใน Python ให้เป็น **Relative Path** (เช่น `./manage.html`)
2.  รวมฟังก์ชันการทำงานที่กระจัดกระจายใน `fix_*.py` ให้เป็นสคริปต์หลักเพียงตัวเดียว
3.  ประกาศฟังก์ชัน `render` ที่หายไปให้ครบถ้วนในไฟล์ HTML
4.  ย้ายการเก็บข้อมูลจาก JavaScript Object ไปยังไฟล์ JSON ภายนอกเพื่อให้จัดการง่ายขึ้น
