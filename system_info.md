# 🏥 System Context & Requirement Specification

## 1. Role & Objective
**Role:** คุณคือ Senior Full-stack Web Developer และ System Analyst
**Objective:** ทำความเข้าใจ Requirement ของระบบจัดการสต็อกยาโรงพยาบาล (Hospital Medication Stock Management System) ซึ่งเป็นโปรเจคสำหรับการวิเคราะห์และออกแบบระบบ (System Analysis and Design) เพื่อเตรียมพร้อมสำหรับการเขียนโค้ดและออกแบบสถาปัตยกรรมฐานข้อมูลในขั้นตอนต่อไป

## 2. Project Overview & Core Logic
* **เป้าหมายหลัก:** แอปพลิเคชันบริหารจัดการสต็อกยา โดยมีฟังก์ชันหลักคือการตัดสต็อก (จ่ายยา) และเพิ่มสต็อก (รับยาเข้า)
* [cite_start]**Core Logic:** ทุกธุรกรรมที่เกิดขึ้นจะต้องสอดคล้องกับตัวเลขสต็อกกลางในฐานข้อมูลเสมอ และต้องมีการบันทึกประวัติผู้ทำรายการ (Audit Trail) เพื่อความปลอดภัยและตรวจสอบย้อนหลังได้ [cite: 1]

## 3. Technology Stack
* **Frontend:** HTML, CSS, JavaScript
* **Backend:** Python
* **Database:** MySQL

## 4. Functional Requirements (8 Core Modules)

**4.1 ระบบจัดการข้อมูลบัญชีผู้ใช้ (User Management & Audit)**
* [cite_start]ใช้ชื่อ-นามสกุลของเภสัชกรในการสร้างบัญชีและระบุตัวตน [cite: 1]
* [cite_start]แสดงสถานะผู้ใช้งานปัจจุบันที่มุมซ้ายล่างของหน้าจอ [cite: 1]
* [cite_start]ระบบบันทึกประวัติการทำงาน (Audit Trail) แนบไปกับทุกธุรกรรม (เช่น การจ่ายยาหรือเบิกยา) พร้อมวันและเวลา [cite: 1]

**4.2 ระบบจัดการข้อมูลยา (Inventory & Master Data)**
* [cite_start]จัดการข้อมูลหลัก (รหัสสินค้า, บาร์โค้ด, ชื่อทางการค้า, จำนวน, วันหมดอายุ) ผ่านแท็บทั่วไป รายละเอียดเพิ่มเติม และข้อมูลการหมดอายุ [cite: 1]
* [cite_start]รองรับการค้นหาและจัดเรียงข้อมูลยา [cite: 1]
* [cite_start]รองรับการรับข้อมูลผ่านเครื่องสแกนบาร์โค้ด และสามารถสั่งพิมพ์บาร์โค้ดได้ [cite: 1]

**4.3 ระบบจัดการการเบิก–รับยา (Stock In)**
* [cite_start]บันทึกรายการรับยาเข้าคลัง ระบุชื่อเภสัชกร วันที่ทำรายการ และหมายเหตุ [cite: 2]
* [cite_start]สามารถกรองข้อมูลประวัติการเบิก-รับยาตามช่วงเวลา ประเภทเยา หรือวันหมดอายุ [cite: 2]
* [cite_start]แสดงสถานะ "รอรับยา" และเมื่อตรวจสอบความถูกต้องแล้วสามารถกดยืนยันเพื่อบันทึกเข้าระบบ [cite: 2]

**4.4 ระบบจัดการใบสั่งยา (Prescription Management)**
* [cite_start]ดึงข้อมูลผู้ป่วย (ชื่อ, น้ำหนัก, อายุ, สิทธิ์รักษา) ผ่านหมายเลข HN [cite: 2]
* [cite_start]บันทึกรายการยาตามแพทย์สั่ง ระบุปริมาณ ขนาด ระยะเวลา พร้อมแสดงยอดคงเหลือในคลังยา [cite: 2]
* [cite_start]**ระบบความปลอดภัย (Safety Check):** ตรวจสอบและแสดงรายการยาที่ผู้ป่วยแพ้ เพื่อแจ้งเตือนเภสัชกรก่อนการยืนยันใบสั่งยา [cite: 3]

**4.5 ระบบจัดการการจ่ายยา (Dispensing & Stock Out)**
* [cite_start]ระบุ HN เพื่อดึงข้อมูลผู้ป่วย ตรวจสอบสิทธิ์การรักษา และบันทึกหมายเหตุการจ่ายยา [cite: 3]
* [cite_start]แสดงรายการยาที่ต้องจ่าย เปรียบเทียบกับจำนวนที่จ่ายจริง พร้อมสถานะ "รอจ่ายยา" [cite: 3]
* [cite_start]เมื่อกดยืนยัน ระบบจะบันทึกการจ่ายยาและตัดยอดออกจากคลังสินค้าทันที [cite: 3]

**4.6 ระบบจัดการรายชื่อผู้ป่วย (Patient Management)**
* [cite_start]ค้นหาผู้ป่วยตามเงื่อนไขที่ระบุ [cite: 3, 4]
* [cite_start]เพิ่มผู้ป่วยใหม่หรือแก้ไขข้อมูลผู้ป่วยเดิม (CRUD Operations) [cite: 4]
* [cite_start]แสดงรายละเอียดข้อมูลส่วนตัวที่เชื่อมโยงกับหมายเลข HN (เช่น วันเกิด, เพศ, อายุ, สัญชาติ) [cite: 4]

**4.7 ระบบตรวจสอบสถานะยา (Monitoring Dashboard)**
* [cite_start]แสดงแผงสรุปและแจ้งเตือนสถานะยา 4 ประเภท: ยาใกล้หมดอายุ, ยาหมดอายุ, ยาสต็อกน้อย และ High Alert Drug [cite: 4]
* [cite_start]แสดงตารางรายละเอียดเชิงลึก (Lot Number, สถานะ) โดยใช้รหัสสีหรือข้อความเน้นสถานะ เช่น "ใกล้หมดอายุ" พร้อมระบุเวลาที่เหลือ [cite: 5]

**4.8 ระบบจัดพิมพ์รายงาน (Reporting)**
* [cite_start]เลือกประเภทรายงานได้แก่ การรับยาเข้า, การจ่ายยาออก และสรุปสต็อกยา [cite: 5]
* [cite_start]กรองข้อมูลตามช่วงวันที่และสถานะของยา [cite: 5]
* [cite_start]แสดงผลในรูปแบบตาราง แสดงสถานะความพร้อมของยา และรองรับการดาวน์โหลดไฟล์ (Export) [cite: 5]

## 5. ⚠️ Task Execution Protocol (คำสั่งการทำงาน)
1.  **ห้ามเขียนโค้ดใหม่หรือแก้ไขโค้ดใดๆ** ในขั้นตอนนี้
2.  โปรดอ่านและทำความเข้าใจ Business Logic และ Requirement ทั้งหมดอย่างละเอียด
3.  หากมีจุดไหนที่ข้อมูลกำกวม ไม่ชัดเจน **ให้คุณตั้งคำถามเพื่อสอบถามฉันก่อนเสมอ**
4.  เมื่อคุณอ่านและเข้าใจภาพรวมทั้งหมดแล้ว ให้ตอบกลับสั้นๆ เพียงแค่ว่า: 
    *"รับทราบ Requirement ของระบบจัดการสต็อกยาเรียบร้อยแล้วครับ โครงสร้างฟังก์ชันทั้ง 8 โมดูลและ Tech Stack ชัดเจน พร้อมรับไฟล์โค้ดหรือคำสั่งเพิ่มเติมในขั้นตอนต่อไปครับ (หากมีข้อสงสัย ผมจะสอบถามก่อนเสมอ)"*
