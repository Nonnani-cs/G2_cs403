# แนวทางการแก้ไขข้อบกพร่องของระบบ (System Remediation Plan)

เอกสารฉบับนี้เสนอแนวทางเชิงเทคนิคในการแก้ไขข้อบกพร่องที่พบในโปรเจกต์ เพื่อให้ระบบมีความเสถียร ปลอดภัย และสามารถนำไปใช้งานต่อได้อย่างยั่งยืน

---

## 1. การจัดการเส้นทางไฟล์ (File Path Remediation)

### ❌ ปัญหา:
ใช้ Absolute Path แบบ Hardcoded (เช่น `/Users/atcharapornn/...`)

### ✅ แนวทางแก้ไข:
เปลี่ยนไปใช้ **Relative Path** โดยอ้างอิงจากตำแหน่งของสคริปต์ Python หรือใช้ Library `os` เพื่อหาตำแหน่งปัจจุบันอัตโนมัติ

**ตัวอย่างโค้ดที่ควรเป็น:**
```python
import os

# หาตำแหน่งโฟลเดอร์ปัจจุบันของสคริปต์
current_dir = os.path.dirname(os.path.abspath(__file__))
filepath = os.path.join(current_dir, 'manage.html')

with open(filepath, 'r', encoding='utf-8') as f:
    # ... ทำงานต่อ ...
```

## 2. การจัดการฟังก์ชันและโครงสร้างโค้ด (Logic & Structure Remediation)

### ❌ ปัญหา:
ฟังก์ชัน `render` หายไป และพารามิเตอร์ไม่สอดคล้องกัน

### ✅ แนวทางแก้ไข:
1.  **Standardize Signatures:** ปรับปรุงฟังก์ชัน `selectDrug` ในทั่งฝั่ง Python และ HTML ให้รับค่าตรงกัน (เช่น `(row, code, name)`)
2.  **Implement Missing Functions:** เขียนฟังก์ชัน `renderPatientTable()` และ `renderInventoryTable()` ลงในไฟล์ HTML ที่เกี่ยวข้อง โดยใช้ Logic วนลูปสร้าง `<tr>` จากข้อมูลใน Array
3.  **Template Engine Approach:** ลดการใช้ Regex ที่ซับซ้อนในการแทนที่ข้อความยาวๆ เปลี่ยนเป็นการใช้ "Marker" สั้นๆ เช่น `<!-- DRUG_LIST_HERE -->` เพื่อให้ Regex ค้นหาและแทนที่ได้แม่นยำขึ้น

---

## 3. ความปลอดภัยและข้อมูล (Security & Data Remediation)

### ❌ ปัญหา:
Session ถูกปลอมแปลงได้ง่าย และข้อมูล MasterData เป็นค่าว่าง

### ✅ แนวทางแก้ไข:
1.  **Basic Protection:** สำหรับระยะสั้น ให้ใช้การเข้ารหัสข้อมูลใน `localStorage` เบื้องต้น หรือย้ายการตรวจสอบสิทธิ์บางส่วนไปไว้ในส่วนที่ยากต่อการเข้าถึงมากขึ้น
2.  **Centralized Data:** สร้างไฟล์ `data.json` ขึ้นมาหนึ่งไฟล์เพื่อเก็บรายชื่อยาและข้อมูลผู้ป่วยทั้งหมด จากนั้นให้สคริปต์ Python ทุกตัวดึงข้อมูลจากไฟล์นี้ที่เดียว (Single Source of Truth) เพื่อลดความซ้ำซ้อน
3.  **Bootstrap Data:** ในไฟล์ HTML ควรมีข้อมูล Mock พื้นฐานอย่างน้อย 1 รายการ เพื่อป้องกันไม่ให้หน้าจอพังเมื่อสคริปต์ Python ยังไม่ได้รัน

---

## 4. การบำรุงรักษาระบบ (Maintenance Remediation)

### ❌ ปัญหา:
สคริปต์กระจัดกระจายและมีหลายเวอร์ชัน (`v1`, `v2`, `v3`)

### ✅ แนวทางแก้ไข:
1.  **Consolidation:** รวมสคริปต์ `replace_drugs_v3.py` และสคริปต์ `fix_*.py` ต่างๆ เข้าเป็นไฟล์หลักไฟล์เดียว (เช่น `update_system.py`) โดยแบ่งเป็นฟังก์ชันย่อยๆ ตามหน้าที่
2.  **Version Clean-up:** ลบไฟล์เวอร์ชันเก่าทิ้งหลังจากรวมโค้ดเสร็จแล้ว เพื่อลดความสับสน
3.  **Documentation:** เพิ่ม Comment ในโค้ด Python ว่าแต่ละส่วนทำหน้าที่แก้ไข HTML บรรทัดไหนหรือส่วนไหน

## ลำดับความสำคัญในการดำเนินการ (Action Priority):
1.  **P0 (ทันที):** แก้ไข Relative Path ในสคริปต์ Python เพื่อให้รันในเครื่อง Windows ได้
2.  **P1 (สำคัญ):** เพิ่มฟังก์ชัน `render` ที่หายไปเพื่อให้หน้าเว็บแสดงข้อมูลได้
3.  **P2 (ปานกลาง):** รวมสคริปต์ Python และจัดระเบียบไฟล์
4.  **P3 (ระยะยาว):** ปรับปรุงระบบ Security และเชื่อมต่อกับ MySQL จริง
