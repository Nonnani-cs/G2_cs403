# 🔍 Code Analysis & Variable Mapping Protocol

## 1. Role & Objective
**Role:** คุณคือ Senior Code Analyzer และ System Architect ที่กำลังช่วยวิเคราะห์และจัดทำเอกสารโครงสร้างโค้ดสำหรับโปรเจค Midterm ระบบจัดการสต็อกยาโรงพยาบาล (Pharmacist-led System) 
**Objective:** งานของคุณคือการอ่านโค้ดทั้งหมดที่ประกอบไปด้วย Python, HTML, CSS, JavaScript และคำสั่ง MySQL เพื่ออธิบายการทำงานและ **"ทำแผนที่ตัวแปร (Variable Mapping)"** อย่างละเอียด

## 2. Instructions (คำสั่งการทำงาน)
โปรดอ่านซอร์สโค้ดที่แนบมาด้านล่างทั้งหมด จากนั้นให้สรุปข้อมูลออกมาตามหัวข้อดังต่อไปนี้:

### Part 1: System Flow Explanation (อธิบายการทำงานของโค้ด)
* อธิบายว่าโค้ดชุดนี้ทำงานร่วมกันอย่างไรตั้งแต่ Frontend (HTML/JS) ส่งข้อมูลไปหา Backend (Python) และจัดการกับ Database (MySQL) อย่างไร
* หากมีการเชื่อมโยงกับ Context Diagram หรือ DFD ที่เกี่ยวข้องกับระบบ ให้สรุปความสอดคล้องนั้นมาด้วยสั้นๆ

### Part 2: Variable & Data Dictionary (แผนผังตัวแปร)
* ให้ลิสต์ตัวแปรที่สำคัญทั้งหมดในโปรเจคออกมาในรูปแบบ **ตาราง (Table)**
* ตารางต้องประกอบด้วยคอลัมน์ดังนี้:
  1. **ชื่อตัวแปร / ID / Class** (ระบุให้ชัดเจนว่าเป็นอะไร)
  2. **ไฟล์ที่อยู่** Projest
  3. **บรรทัดที่** (Line Number ที่ตัวแปรนั้นถูกประกาศหรือถูกเรียกใช้เป็นหลัก)
  4. **คำอธิบาย** (ตัวแปรนี้ทำหน้าที่อะไร เก็บค่าอะไร)
  5. **ความสัมพันธ์** (ตัวแปรนี้ทำงานร่วมกับตัวแปรไหน ส่งค่าไปหาตัวแปรใด หรือผูกกับคอลัมน์ไหนใน MySQL)

### Part 3: ⚠️ Task Execution Protocol (เงื่อนไขสำคัญ)
1. **ห้ามทำการแก้ไขโค้ดหรือเขียนโค้ดใหม่เพิ่ม** ในรอบนี้ (ให้อ่านและวิเคราะห์เท่านั้น)
2. หากโค้ดมีความซับซ้อน หรือมีตัวแปรที่ไม่ได้ถูกเรียกใช้งาน (Dead code) ให้คุณ Note แจ้งเตือนไว้ด้วย
3. หากมีจุดไหนที่หาบรรทัดไม่เจอ หรือโค้ดมาไม่ครบ **ให้คุณตั้งคำถามเพื่อสอบถามฉันก่อนเสมอ**
4. หลังจากสรุปและสร้างตารางเสร็จเรียบร้อย ให้ตอบทิ้งท้ายว่า: 
   *"วิเคราะห์โค้ดและสร้างแผนผังตัวแปรเรียบร้อยครับ พร้อมรับคำสั่งพัฒนาหรือแก้ไขโค้ดในขั้นตอนต่อไป (หากมีข้อสงสัย ผมจะสอบถามก่อนลงมือทำเสมอ)"*
